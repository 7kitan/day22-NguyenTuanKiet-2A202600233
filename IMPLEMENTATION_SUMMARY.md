## Key Implementation Details

### Step 1: LangSmith RAG Pipeline
- Uses `@traceable` decorator to send traces to LangSmith
- Builds FAISS vectorstore from knowledge base
- Runs 50 questions through RAG chain
- Each query creates a trace visible in smith.langchain.com

### Step 2: Prompt Hub & A/B Routing
- Defines 2 distinct prompts (V1: concise, V2: structured)
- Pushes both to LangSmith Prompt Hub
- Deterministic routing: `hash(request_id) % 2 → V1 or V2`
- Logs which version handled each request

### Step 3: RAGAS Evaluation
- Runs all 50 QA pairs through both prompt versions
- Captures retrieved contexts as list[str] (required by RAGAS)
- Evaluates with 4 metrics using LLM-based scoring
- Compares V1 vs V2 performance
- Saves results to JSON for submission

### Step 4: Guardrails Validators
- **PIIDetector**: Regex patterns for EMAIL, PHONE, SSN, CREDIT_CARD
- **JSONFormatter**: Repairs markdown fences, single quotes, trailing commas
- Both use `PassResult` for successful validation/repair
- Demonstrates on 6 PII cases + 5 JSON cases
